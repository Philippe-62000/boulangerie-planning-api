Délégation du login partenaire à l’API Render (remplace la route Mongo qui loggue « [partner-auth-login] »)

--- Étape A — Trouver l’ancienne route dans le projet Next.js (dossier Vercel, pas l’API Render) ---

1) Ouvrir le repo du site commande-longuenesse dans Cursor.

2) Recherche globale (Ctrl+Shift+F) : partner-auth-login
   ou : partner-auth

3) Fichiers à remplacer en priorité :
   App Router  : app/api/partner-auth/login/route.ts  (+ me/route.ts si présent)
   Pages Router: pages/api/partner-auth/login.ts      (+ me.ts si présent)

4) Ne pas garder deux implémentations pour la même URL : si app/api et pages/api
   existent tous les deux pour partner-auth, supprimer l’une des deux versions.

--- Étape B — Copier les fichiers proxy depuis CE dépôt (boulangerie-planning) ---

App Router :
  Copier le dossier :
    partner-commande-vercel-drop-in/app-router/app/api/partner-auth/
  Vers votre projet Vercel :
    app/api/partner-auth/
  (login/route.ts et me/route.ts)

Pages Router :
  Copier le dossier :
    partner-commande-vercel-drop-in/pages-router/pages/api/partner-auth/
  Vers votre projet Vercel :
    pages/api/partner-auth/

--- Étape C — Variables Vercel ---

PARTNER_API_UPSTREAM = URL de base de l’API Render, SANS slash final
Exemple : https://boulangerie-planning-api-xxx.onrender.com

--- Étape D — Déployer ---

Commit, push, redeploy Vercel. Les logs Vercel ne devraient plus montrer « introuvable »
côté Mongo local pour ce login : la route relaie vers Render.

--- Suite (commandes / JWT) ---

Les appels suivants (partner-orders, etc.) doivent soit utiliser la même JWT_SECRET
que Render sur Vercel si la vérif est locale, soit appeler les routes Render avec le token.

Si vous gardez Mongo direct sur Vercel au lieu du proxy : MONGODB_URI identique à Render,
collection « partnercompanies », champs site et passwordHash après déploiement API Render.

--- Mini-viennoiseries petit déjeuner (2026) ---

1) Admin Filmara : onglet Formules → sous la description, choisir 1 / 2 / 3 mini-viennoiseries par formule.

2) Site entreprise Vercel :
   - GET /api/partner-orders/formulas (token entreprise) pour lire miniViennoiserieCountPerFormula.
   - Dans le formulaire : nombre de formules (quantity) + composant
     partner-commande-vercel-drop-in/components/PartnerBreakfastMiniViennoiseries.jsx
   - POST /api/partner-orders/my avec body :
     { fulfillment, datetime, mealType, tier, quantity, miniViennoiserieDetail: [{name, quantity}] }
   - Utiliser canSubmitBreakfastOrder() pour bloquer l’envoi si le total ne colle pas.

3) Suppression commande (Filmara + Vercel si même API Render) :
   - DELETE /api/partner-orders/internal/:id (salarié Filmara)
   - DELETE /api/partner-admin/orders/:id (admin)
   - DELETE /api/partner-orders/my/:id (entreprise, ses commandes seulement)
   Si Vercel a une copie Mongo locale : ajouter action deleteOrder dans internal-partner-sync
   (voir app-router/app/api/internal-partner-sync/route.ts.example).
